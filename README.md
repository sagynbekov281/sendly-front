# Sendly — заявки на перевод денег между друзьями

Веб-приложение на React + TypeScript + Tailwind CSS. Позволяет регистрироваться, добавлять контакты (друг / муж-жена / семья / коллега) и отправлять заявки вида «скинь мне денег» с суммой и сообщением. Получатель видит уведомление, может принять/отклонить и оплатить заявку прямо в приложении.

## Стек

- **React 18** + **TypeScript**
- **React Router v6** — навигация
- **Tailwind CSS** — стили, адаптив под мобильные
- **Zustand** — состояние на клиенте
- **react-icons** (Remix Icon набор)
- **Vite** — сборка

## Backend

Этот фронт работает с отдельным backend-проектом (`sendly-backend`, Node.js + Express + PostgreSQL). Все данные — пользователи, контакты, заявки, уведомления — хранятся в настоящей базе данных, поэтому работают между любыми устройствами и браузерами.

### Подключение к backend

1. Задеплой `sendly-backend` (см. его README — там инструкции под Render/Railway).
2. Скопируй `.env.example` в `.env.local`:
   ```bash
   cp .env.example .env.local
   ```
3. Впиши туда URL своего backend:
   ```
   VITE_API_URL=https://your-backend.up.railway.app
   ```
4. Для локальной разработки можно держать backend локально (`npm run dev` в папке backend на порту 4000) и оставить `VITE_API_URL=http://localhost:4000`.

## Запуск проекта

```bash
npm install
npm run dev
```

Откроется на `http://localhost:5173`.

Для продакшен-сборки:

```bash
npm run build
npm run preview
```

## Функционал

1. **Регистрация / Вход** — никнейм (уникальный, валидируется), отображаемое имя, email, пароль.
2. **Контакты** — добавление пользователя по никнейму с категорией: Друг / Муж-Жена / Семья / Коллега.
3. **Заявка "скинь мне деньги"** — указываешь никнейм получателя, категорию отношений, сумму (есть быстрые кнопки 500/1000/2000/5000), сообщение.
4. **Уведомления** — получатель видит, что ему прислали заявку; отправитель видит, когда заявку приняли/отклонили/оплатили.
5. **Оплата** — получатель заявки нажимает «Оплатить», деньги списываются с его баланса и зачисляются на баланс того, кто попросил (упрощённая модель оплаты, без реального процессинга — место, куда подключается платёжный backend).
6. **Адаптив** — на десктопе боковое меню, на мобильном — нижняя навигационная панель.
